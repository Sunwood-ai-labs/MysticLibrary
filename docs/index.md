## 目的

- プロンプトの属人化を減らす
- 再利用しやすいテンプレートを提供する
- 更新履歴と運用ルールを明確にする

## 使い方（最短）

1. [`プロンプトカタログ`](/prompt-catalog/) からカテゴリを選ぶ
2. 個別ページの `intent` / `input_requirements` を確認する
3. 入力例を参考に必要情報を埋める
4. 出力例と注意点を確認して実運用へ載せる

## 対象読者

- 社内で LLM を使った文章作成・開発支援を行うメンバー
- チームでプロンプトをレビュー・共有したい運用担当
