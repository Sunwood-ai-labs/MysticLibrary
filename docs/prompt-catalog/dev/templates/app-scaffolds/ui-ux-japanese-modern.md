# にアクセスし、以下の要件をすべて満たす作業を

https://t.co/Lj1Q7ca46C

## プロンプト本文
~~~text
https://t.co/Lj1Q7ca46C
にアクセスし、以下の要件をすべて満たす作業を
**最初から最後まで一連のタスクとして実行してください。**

【作成するもの】
モダンでオシャレなデザインと優れたUI/UXを兼ね備えた
「日本風おみくじアプリ」

和風モダン(Japanese Modern)をテーマに、
・落ち着いた配色
・和柄や縦書きを想起させるデザイン
・直感的で気持ちよく使えるUI/UX
を意識してください。

【必須成果物(省略不可)】
1. 動作する日本風おみくじアプリ
2. GitHub にデプロイされたリポジトリ
3. https://t.co/lPlBAYWdkG(日本語)
4. README_EN.md(英語)
5. アプリの Share URL(公開URL)

【Share URL の取り扱い(重要・初期要件)】
- 取得した Share URL は以下 **2 箇所に必ず反映**してください。

1 GitHub リポジトリ設定 
- リポジトリの「About」セクションにある 
 **Website(Web)欄** に Share URL を設定すること。

2 README ファイル 
- `https://t.co/lPlBAYWdkG` および `README_EN.md` の両方に、
 Live Demo URL として明示的に記載すること。

【README 作成要件(初期要件)】
README はアプリ開発と同時に整備するようにgemini ai studioでGeminiに指示してください。

### https://t.co/lPlBAYWdkG(日本語)
- 日本語で記述
- 日本向けプロダクト/ポートフォリオ用途
- 高い完成度のプロフェッショナルな内容
- Markdown 装飾を積極的に使用
- 冒頭または目立つ位置に Live Demo URL を記載

### README_EN.md(英語)
- 英語で記述
- 海外エンジニア・プロダクト紹介向け
- 自然な英語表現(直訳禁止)
- Markdown 装飾を積極的に使用
- 冒頭または目立つ位置に Live Demo URL を記載

### 両 README 共通の必須構成
1. アプリ概要 / Overview 
2. 特徴 / Features 
3. 使用技術 / Tech Stack 
4. 使い方 / How to Use 
5. Live Demo URL 
6. 今後の展望 / Future Improvements 

※ https://t.co/lPlBAYWdkG の冒頭には必ず以下を記載してください。 
「👉 English version: README_EN.md」

【実行手順と操作の厳守ルール】
以下は **順番厳守** です。

1. アプリ構築完了後も、アプリ一覧(Your apps)に戻らないこと。
2. 編集画面右上の「GitHub」ボタンを押し、必ずデプロイすること。
3. 次に「Share app」をクリックすること。
 - URL は必ず「Copy」ボタンで取得すること。
4. 取得した Share URL を 
 - GitHub リポジトリの Website 欄 
 - https://t.co/lPlBAYWdkG / README_EN.md 
 のすべてに反映すること。
5. 上記すべてが完了した後、
 - Share URL
 - https://t.co/lPlBAYWdkG の全文
 - README_EN.md の全文
 を最終結果として提示すること。

【完了条件】
- GitHub リポジトリの Website 欄に Share URL が設定されていること
- https://t.co/lPlBAYWdkG / README_EN.md に Live Demo URL が記載されていること
- 共有URLと README の全文が提示されていること

上記すべてを満たした時点でタスク完了とします。
~~~
