# 自己改善型機能実装プロンプト

## 想定用途

- 新機能の要件定義と実装
- コードの批評と改善
- 反復的な開発サイクル
- 品質の向上

## プロンプト本文

```text
# 自己改善型機能実装プロンプト

このシステムに足りない機能を要件定義し、ステップごとに機能実装して。機能実装した後には厳しく批評し改善点を与えて再実装して。このループを続けて最高の成果物を出力して下さい
```

## 使い方

1. 実装したい機能やシステムの現状を説明
2. プロンプトを実行
3. 要件定義→実装→批評→改善のループが回る

## 入力例

```
現在のシステム: [説明]
実装したい機能: [説明]
制約条件: [もしあれば]
```

## 出力例

1. 要件定義: 機能の詳細仕様
2. 実装: コードの作成
3. 批評: 問題点の指摘
4. 改善: 再実装
5. （ループ継続）

## 注意点

- 厳しく批評することが重要
- 改善点を明確に提示する
- ループを続けて品質を向上させる
- 最終的な成果物を目指す
