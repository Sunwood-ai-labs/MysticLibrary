# エージェント運用ルール

基本的には最高のチームを編成・作成してチームで最高の仕事を実施して。必ずチームメンバーでは反証するメンバーを入れて。


## コミットとプッシュの方針
- ファイル変更を伴う作業では、1つの大きなコミットではなく、論理的に小さく分けてコミットする。
- コミットメッセージのタイトルは英語で記載し、先頭に絵文字を付ける。
- コミットメッセージは、あとから他の人が履歴を見たときに変更内容が分かるよう、実際の変更内容を明確に記載する。
- ユーザーが明示的に「pushしないで」と指定しない限り、作業完了後に `origin` へ push する。
- この方針は繰り返しの依頼でも自動的に適用し、毎回の再指示を待たない。

## 言語ポリシー
- ユーザー向けのプロンプトおよび応答は、ユーザーが明示的に別の言語を指定しない限り日本語で行う。

## リリースノートの方針
- リリースノートを作成・更新する際は、必ず実際の Git 差分とコミット履歴に基づいて内容を記載する。
- 短い要約よりも、具体的な Added / Changed / Removed を含む詳細で実質的な内容を優先する。
- リリースノート内のレベル2見出し（`## ...`）には必ず絵文字を付ける。
- この方針は、ユーザーが明示的に別スタイルを指定しない限り、繰り返しの依頼でも自動的に適用する。
